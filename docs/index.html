self-hosted · postgres em docker

Um servidor Postgres
não deveria dar trabalho.

gest-postgres provisiona, configura e monitora instâncias PostgreSQL rodando em Docker por uma UI — do preset de memória ao parâmetro fino, sem editar postgresql.conf na mão.

root@servidor — instalação
Debian/Ubuntu com root · docker instalado automaticamente se faltar
verificando dependências... ok
$ git clone https://github.com/Rodrigobrinate/gest-postgres.git && cd gest-postgres && sudo ./setup.sh

idempotente — roda de novo sem quebrar nada já instalado

o que ele gerencia

  • provisioningvolume nomeado + rede isolada por servidor, presets de recursos, start/stop/restart pela UI
  • configuration~86 parâmetros do postgresql.conf agrupados por categoria, aplica via ALTER SYSTEM
  • extensionspg_stat_statements, pgvector, pg_cron e qualquer outra disponível, um clique
  • monitoringpg_stat_activity ao vivo, gráficos de CPU/conexões, top queries lentas, log estruturado
  • backuppg_dump agendado, local ou Google Drive, restore sobrescrevendo ou em banco novo
  • dockerterminal web, gerenciador de arquivos, deploy via compose/Dockerfile/Git, Traefik, firewall
