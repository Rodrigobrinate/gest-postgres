self-hosted · Docker · open source

Gerencie servidores PostgreSQL
em Docker, visualmente

Provisiona instâncias, ajusta config (do preset simples ao parâmetro avançado), habilita extensões, monitora métricas e agenda backups — tudo por uma UI, sem tocar em SQL na mão.

git clone https://github.com/Rodrigobrinate/gest-postgres.git && cd gest-postgres && sudo ./setup.sh

Debian/Ubuntu com root · instala Docker sozinho · idempotente, pode rodar de novo

Provisionamento

Cria servidor com volume + rede isolada, presets de recursos (Pequeno/Médio/Grande), start/stop/restart/excluir pela UI.

Configuração visual

~86 parâmetros do postgresql.conf agrupados por categoria, aplica via ALTER SYSTEM + reload, sem editar arquivo.

Extensões

pg_stat_statements, uuid-ossp, pgcrypto, pg_trgm, pgvector, pg_cron e qualquer outra disponível — um clique.

Monitoramento

pg_stat_activity ao vivo, gráficos de conexões/CPU/memória, top queries lentas, log tail.

Backup / Restore

pg_dump agendado (local ou Google Drive), retenção configurável, restore sobrescrevendo ou em banco novo.

Gestão Docker genérica

Terminal web em qualquer container, file manager, deploy via compose/Dockerfile/Git com webhook, Traefik, firewall.
